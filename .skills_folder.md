# 📂 Skill Folder Architecture

This document serves as the structural reference for creating self-contained AI capability modules. Adhering to this hierarchy ensures optimal discovery, execution, and resource management.

## 🏗️ Hierarchy Overview

A skill must reside in a dedicated directory named using `kebab-case`. The internal structure is organized by functional role:

```text
[skill-name]/
├── SKILL.md          # Control Plane (Metadata & Instructions)
├── scripts/          # Execution Layer (Executable code)
├── references/       # Knowledge Layer (Static documentation)
└── assets/           # Resource Layer (Templates & static files)

## 🏗️ Hierarchy Overview
A skill must reside in a dedicated directory named using `kebab-case`. The internal structure is organized by functional role:

```text
[skill-name]/
├── SKILL.md          # Control Plane (Metadata & Instructions)
├── scripts/          # Execution Layer (Executable code)
├── references/       # Knowledge Layer (Static documentation)
└── assets/           # Resource Layer (Templates & static files)

------------------------------
🛠️ Component Definitions1. SKILL.md (The Control Plane)
The mandatory entry point for every skill. It uses YAML frontmatter to register the skill with the system.

* Metadata: Contains the name (unique ID) and description. The description is the semantic trigger used by the LLM to determine task relevance.
* Workflow: The body contains the specific instructions, logic flow, and guardrails the agent must follow when the skill is active.

2. /scripts (The Execution Layer)
Houses deterministic logic that extends beyond natural language processing.

* Role: Contains Python, Bash, or Node scripts.
* Usage: Use for data transformation, API calls, or environmental configurations that require exact execution.

3. /references (The Knowledge Layer)
A repository for deep context and static information.

* Role: Contains API documentation, style guides, or domain-specific whitepapers.
* Optimization: Keeps the primary SKILL.md lean. The agent retrieves these files only when specific domain knowledge is required.

4. /assets (The Resource Layer)
Contains supporting static files used during skill execution.

* Role: Stores boilerplate templates, JSON schemas, images, or datasets.
* Usage: Provides the raw materials for generating reports or validating user inputs.

------------------------------
🚀 Operational Standards

   1. Discovery: Skills are loaded from ~/.claude/skills/ (Global) or ./.claude/skills/ (Project-specific).
   2. Persistence: The SKILL.md is parsed on startup. Changes to the directory structure require a session refresh.
   3. Efficiency: Instructions should prioritize "Progressive Disclosure"—only loading deep reference data when the immediate task demands it.
